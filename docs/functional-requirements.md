# Функциональные требования (реалистичные)

## Контекст
Система учета и управления процессами в клинике sleeving. 4 роли: Meth (клиент), Sleeve Broker, Needlecaster, Psychosurgeon. Учет клиентов, заказов, инвентаря, статусов процедур.

## Требования

### FR-001: Каталог sleeves
Система должна вести инвентарь доступных тел (sleeves) с атрибутами: ID, рост, вес, пол, возраст, физический тип, дата поступления, текущий статус (свободен/зарезервирован/в подготовке/готов к использованию/списан). Sleeve Broker должен иметь возможность фильтровать и сортировать по любым параметрам.

### FR-002: Управление культивированием
Для заказных sleeves система должна отслеживать статус культивирования: дата начала, выбранный донор ДНК, текущая стадия (в неделях), плановая дата готовности. При изменении статуса система автоматически уведомляет Sleeve Broker и Meth (email/SMS).

### FR-003: Резервирование sleeves
Sleeve Broker должен иметь возможность зарезервировать конкретный sleeve за конкретным Meth на определенный период. Система должна блокировать повторное резервирование и автоматически снимать резервирование через 72 часа, если sleeve не перешел в статус "в подготовке".

### FR-004: Медицинская история sleeves
Для каждого sleeve система должна хранить медицинскую карту: результаты анализов крови, ЭКГ, кардиограммы, даты обследований, выявленные проблемы, проведенные процедуры. Доступ к карте — только у Sleeve Broker и назначенного врача.

### FR-005: Планирование операций
Система должна отображать календарь операций с временными слотами. Needlecaster видит свою очередь на извлечение стеков и needlecast. Система проверяет конфликты (нельзя назначить две операции в один слот) и отправляет уведомления за 24 часа до процедуры.

### FR-006: Учет кортикальных стеков
Система должна вести учет всех стеков в клинике: ID стека, владелец (Meth), текущее местоположение (в теле/в хранилище/в процессе переноса), дата последнего извлечения, статус (активен/архивирован/поврежден). История перемещений — кто, когда, из пункта A в пункт B.

### FR-007: Отслеживание процедуры needlecast
Система должна фиксировать этапы процедуры переноса: время начала, фазы (моторика/сенсорика/вегетативные функции/сознание), время пробуждения, ответственный Needlecaster. Если процедура длится дольше нормы (>6 часов), система отправляет alert руководству.

### FR-008: Тестирование после пробуждения
После needlecast Needlecaster должен фиксировать в системе результаты первичного тестирования: время пробуждения, ответы Meth на контрольные вопросы (год, личность, последнее воспоминание), статус (пройден/не пройден). При непрохождении система блокирует переход к валидации.

### FR-009: 72-часовое наблюдение
Система должна вести таймер валидации с чекпоинтами: 24 часа (первичные тесты), 48 часов (психологическая оценка), 72 часа (финальная сертификация). Psychosurgeon отмечает прохождение каждого этапа. Система отправляет напоминания при приближении дедлайнов.

### FR-010: Сертификация совместимости
При успешном прохождении 72-часовой валидации система должна автоматически генерировать сертификат психосоматической совместимости в PDF с реквизитами клиники, ID клиента, датами процедур и QR-кодом для проверки подлинности.

### FR-011: Диагностика проблем
Если в процессе валидации выявлены проблемы (stack shock, диссонанс, отторжение), Psychosurgeon фиксирует диагноз в системе. Система автоматически создает инцидент с рекомендациями: корректировка стека/смена sleeve/усиленная терапия, и блокирует выпуск клиента до разрешения.

### FR-012: Журнал операций со стеками
Система должна вести неизменяемый журнал (audit log) всех операций: извлечение (дата, оператор, ID стека), перемещение между секторами, временное хранение, needlecast (целевой sleeve). Лог должен быть доступен только директору клиники для аудита безопасности.

### FR-013: Личный кабинет Meth
Meth должен иметь доступ к порталу с информацией о своем кейсе: текущий статус (ожидание sleeve/готов к извлечению/процедура назначена/в процессе/валидация/завершено), даты назначенных процедур, контакты ответственных специалистов, скачивание сертификатов и документов.
